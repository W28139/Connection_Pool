# MySQL 数据库连接池

本项目是一个基于 C++11 实现的高性能 MySQL 数据库连接池。通过引入**生产者-消费者模型**、**单例模式**及**长连接复用机制**，显著提升了高并发场景下数据库操作的吞吐量，并有效降低了系统资源的消耗。

## 1. 核心设计原理

### 1.1 连接复用
在传统的短连接模式下，每次数据库操作都需要经历 `TCP三次握手` -> `MySQL身份验证` -> `执行SQL` -> `TCP四次挥手`。在高并发场景下，频繁的连接建立与销毁会产生巨大的网络开销和 CPU 损耗。连接池通过预先创建并维护一定数量的“长连接”，让多个线程可以循环复用这些连接，消灭了建连成本。

### 1.2 生产者-消费者模型
*   **生产者线程**：负责实时监控连接池中空闲连接的数量。当连接数不足且未达到上限时，主动创建新连接。
*   **消费者**：通过接口从连接池获取连接。
*   **等待通知机制**：使用 `std::condition_variable` 实现线程间的同步，避免了忙轮询导致的 CPU 浪费。

### 1.3 资源自动管理
利用 C++11 的智能指针 `std::shared_ptr` 并配合**自定义删除器**。当用户使用完连接后，智能指针析构时不会关闭底层 MySQL 连接，而是自动将其重新放回连接池队列中。

## 2. 系统级优化策略

为了获得最真实的连接性能对比，本项目在压测过程中对操作系统和数据库进行了深度调优，确保瓶颈聚焦在“连接过程”本身。

### 2.1 磁盘 I/O 优化
默认情况下，MySQL 每插入一条数据都会强制刷盘，这会导致 IO 阻塞，掩盖连接池的优势。
*   **MySQL 配置**：设置 `innodb_flush_log_at_trx_commit = 2`。这让 MySQL 每秒刷盘一次，而非实时刷盘，极大地提升了写入速度。
*   **应用层优化**：引入**显式事务**。将大批量数据打包成一个事务提交，将磁盘物理写入频率降至最低。

### 2.2 TCP 网络协议栈优化
短连接模式在极短时间内会产生海量的 `TIME_WAIT` 状态连接，导致可用端口耗尽。
*   **内核参数**：开启 `net.ipv4.tcp_tw_reuse = 1`，允许系统快速回收并重用处于 `TIME_WAIT` 的端口，确保 64 线程并发压测不崩溃。

## 3. 压力测试报告

### 3.1 测试环境
*   **硬件**：拯救者 Y7000P (Intel i7-14700H / RTX 4070 / 16G DDR5)
*   **系统**：Ubuntu 22.04 LTS
*   **数据库**：MySQL 8.0
*   **数据量**：10,000 条记录插入

### 3.2 压测数据对比

| 并发线程数 | 不使用连接池| 使用连接池| **性能提升倍数** |
| :--- | :--- | :--- | :--- |
| **1 线程** | 259,130 ms | 65,687 ms | **3.94 倍** |
| **8 线程** | 156,440 ms | 13,707 ms | **11.41 倍** |
| **16 线程** | 164,859 ms | 9,353 ms | **17.62 倍** |
| **32 线程** | 194,310 ms | 4,859 ms | **40.01 倍** |
| **64 线程** | 234,310 ms | 2,282 ms | **102.66 倍** |

### 3.3 结论分析
*   **低并发下**：连接池通过省去 TCP 握手时间，获得了约 4 倍的平稳提升。
*   **高并发下**：不使用连接池的方案由于严重的内核态上下文切换和网络握手排队，耗时剧增至 234 秒；而连接池方案凭借高效的连接复用，仅需 2.2 秒即可完成。**性能差距达到了的 100 倍以上。**
*   **稳定性**：连接池方案在并发增加时，吞吐量保持线性增长，未出现连接超时或系统崩溃现象，展现了极强的工业级稳定性。

## 4. 快速开始

### 编译
需要系统中已安装 `libmysqlclient` 开发库。
```bash
g++ -o press_test stress_test.cpp \
    ./src/CommonConnectionPool.cpp \
    ./src/Connection.cpp \
    -I./include \
    -lmysqlclient -lpthread -std=c++11
```

### 配置文件 (mysql.cnf)
```ini
ip = 127.0.0.1
port = 3306
username = root
password = 123456
dbname = db_proxy
initSize = 10
maxSize = 100
maxIdleTime = 60
ConnectionTimeOut = 5000
```

### 运行自动化压测
```bash
chmod +x run_test.sh
./run_test.sh
```
